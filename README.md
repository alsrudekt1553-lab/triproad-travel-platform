# TRIPROAD

국내 여행 패키지 예약 플랫폼 프로젝트입니다.

## 프로젝트 소개

사용자가 여행 상품을 조회하고 예약, 결제, 후기 작성까지 진행할 수 있는 여행 패키지 예약 서비스를 구현한 팀 프로젝트입니다.

## 기술 스택

### Frontend

* React
* TypeScript
* Axios

### Backend

* Java
* Spring Boot
* JPA
* Oracle Database

## 담당 기능

* 마이페이지 메인
* 예약 내역
* 여행 캘린더
* 체크리스트
* 찜목록
* 나의 후기
* 나의 문의
* 회원정보 수정

## 주요 구현 내용

* REST API 연동
* 사용자별 예약 데이터 조회
* 여행 일정 캘린더 구현
* 체크리스트 CRUD 기능 구현
* 사용자 중심 마이페이지 기능 개발

## 프로젝트를 통해 배운 점

1차 프로젝트를 통해 시스템 내부의 동작 방식을 경험했다면, 2차 프로젝트에서는 사용자의 입장에서 서비스를 바라보는 것을 목표로 프로젝트를 진행하였습니다.

여행 패키지 예약 서비스의 마이페이지 기능을 담당하며 사용자가 어떤 정보를 필요로 하는지, 어떤 방식으로 보여주는 것이 더 편리한지 고민하였습니다.

예약 상태를 기반으로 후기 작성 여부를 판단하거나, 별도의 상태값 없이 여행 날짜를 활용해 캘린더에서 여행 전·여행 중·여행 완료를 구분하는 기능 등을 구현하였습니다.

또한 체크리스트와 같은 사용자 중심 기능을 추가하며 기능 구현뿐만 아니라 사용 경험까지 고려하는 과정이 즐거웠습니다.

비록 AI의 도움을 받아 개발을 진행한 부분도 있었지만, API를 설계하고 데이터를 연결하여 화면에 보여주는 전체 흐름을 이해하려고 노력하였으며, 이를 통해 웹 서비스가 동작하는 구조를 더욱 깊이 이해할 수 있었습니다.

## Trouble Shooting

### 사용자 예약 데이터 조회

초기에는 사용자별 예약 데이터를 정상적으로 조회하지 못하는 문제가 발생하였습니다.

API 응답 데이터와 프론트엔드 타입 정의를 점검하며 데이터 구조를 수정하였고, 이를 통해 사용자별 예약 정보를 정상적으로 조회할 수 있도록 개선하였습니다.

### 여행 상태 표시 기능

여행 전, 여행 중, 여행 완료에 대한 별도의 상태값이 존재하지 않는 문제가 있었습니다.

이를 해결하기 위해 사용자가 예약한 여행 날짜를 기준으로 현재 날짜와 비교하여 상태를 계산하도록 구현하였고, 캘린더에서 시각적으로 확인할 수 있도록 개선하였습니다.
